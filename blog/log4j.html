Log4j worldwide vulnerability

18.12.2021

Last week a critical vulnerability was found in one of the most popularly used logging libraries...

What is Log4j?

Apache Log4j is a Java-based logging utility developed by the Apache Software Foundation. Log4j library has a worldwide usage which enables firms to log data and configure a wide set of applications. The vulnerability which was found lets hackers to inject and run any code on the instances where this Log4j framework is used.

MITRE has rated the vulnerability as critical and assigned a CVSS (Common Vulnerability Scoring System ) score of 10/10 (CVE-2021-44228 ).

What can happen?

This vulnerability, also known as Log4Shell, allows remote code execution in many applications through web requests and without authentication, which enables unlimited access to the information and operational technology infrastructure.

This may result in:
  • Disruption of companies' operations

  • Information theft and takedown of services

  • Reputational damage

  • Unbearable recovery costs

  • Tracking and disclosing the affected user data


How to avoid this?

  • Isolate all Log4j usages across the assets and identify any other connections

  • Prioritize the most important priorities and work on mitigation strategies

  • Modify firewall and put in place suitable rules and prevention mechanisms

  • Make a plan for the worst case scenario and allocate extra people to react in case of any attack


Conclusion

It is clear that there will always be new vulnerabilities and nothing can never be 100% secure. This is a great example that worldwide vulnerabilities still exist which could end with a zero-day. Hopefully now organizations think more about cyber security and will prevent this from happening in the future.
